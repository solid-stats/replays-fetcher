# Phase quick-260623-x57 Plan 01: Skip per-row detail fetch in watch discovery — Summary

Moved the dedup gate AHEAD of the per-row detail fetch in watch discovery: a list row whose
trustworthy `externalId` is already staged is now skipped before any detail HTML is fetched (no
`fetchText` call, no request-spacing slot), collapsing steady-state watch cycles from ~15-20s to
roughly (new rows × spacing) + one list read. The optimization is watch-only (predicate injected);
run-once / `discover --dry-run` pass no predicate and are byte-for-byte unchanged. The skip is
surfaced as a distinct `skippedPreDetail` count carried discovery report → run summary, and the
post-fetch DEDUP-01 gate in `ingestPage` stays intact as defense-in-depth.
