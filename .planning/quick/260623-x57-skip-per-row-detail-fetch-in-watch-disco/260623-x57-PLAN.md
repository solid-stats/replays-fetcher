<objective>
Move the dedup gate AHEAD of the per-row detail fetch in watch discovery. Today the watch
cycle fetches a detail HTML page for every page-1 list row (serialized under the request-spacing
floor) only to extract a filename, then DEDUP-01 in `ingestPage` skips the byte download for
rows already staged — so the detail fetch for an already-staged replay is wasted work that
dominates per-cycle wall-clock (~15-20s for a 60-80 row page).

This plan threads an OPTIONAL `existsBySourceIdentity` predicate (+ `sourceSystem`) into discovery
and gates the row loop in `discoverPageCandidates`: a row whose trustworthy `externalId` is already
staged is skipped before any detail fetch. run-once and `discover --dry-run` pass no predicate and
stay byte-for-byte unchanged. A distinct `skippedPreDetail` tally carries the skip through
discovery → watch-loop → run summary (§AA legibility). The post-fetch DEDUP-01 gate stays as
defense-in-depth for derived-id rows and cross-cycle races.

Purpose: collapse steady-state watch cycles from ~15-20s to sub-second (only NEW rows pay a
detail fetch + spacing gap).
Output: a pre-detail watch-only dedup gate, an observable distinct skip count, and updated/new
discovery + watch tests; verify green at 100% coverage with band-fences intact.
</objective>
