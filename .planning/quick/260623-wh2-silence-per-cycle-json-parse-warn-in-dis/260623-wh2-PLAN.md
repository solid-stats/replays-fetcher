<objective>
Silence the per-cycle `fixture JSON parse failed` warn that floods Sentry from the watch loop
and run-once cron (regression from commit `4ba0613`). The production source (sg.zone) returns
HTML, so `parseSourceFixture` runs `JSON.parse` on HTML every discovery cycle (~1/17s), throws a
`SyntaxError`, warns, then falls back to HTML discovery (which works). The warn is noise for the
expected HTML path; it must fire ONLY for a genuinely-malformed JSON body.

Purpose: Stop drowning real Sentry signal with an expected, recurring SyntaxError while keeping
the §AA log-diagnosability guarantee for the one case that is a real anomaly (a body that looked
like JSON but failed to parse).
Output: A cheap pre-parse sniff in `parseSourceFixture`, plus a flipped test that no longer
encodes the bug.
