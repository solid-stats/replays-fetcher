## Critical Gaps

### RUN-04: Connectivity Check Is Not Implemented

`src/cli.ts` reports:

- `sourceConnectivity: "not-implemented"`
- `s3Connectivity: "not-implemented"`
- `stagingConnectivity: "not-implemented"`

while returning `ok: true` for valid configuration. This does not meet the operator flow promised by RUN-04: a config/connectivity check mode for source, S3, and staging database settings.

### INT-04 / STAGE-01 / STAGE-03: Discovered Timestamp Evidence Is Dropped

Discovery can parse `metadata.discoveredAt`, but the value is not propagated into raw storage evidence or staging payloads. `src/staging/payload.ts` preserves `fetchedAt` in `promotionEvidence`, and `src/staging/postgres-staging-repository.ts` inserts `payload.replayTimestamp`, but the mapper never sets `replayTimestamp` from discovery evidence.

This weakens the staging handoff contract that requires source identity, source URL, object key, checksum, byte size, discovered/fetched timestamps, and status evidence.
