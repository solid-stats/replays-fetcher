<objective>
Create the cross-cutting typed error base class `AppError` in a new `src/errors/` directory. This is the foundation CORE-01 requires: a single generic abstract error class that all domain/operational errors extend, preserving native ES2022 `cause`, a stable per-subclass literal `code`, an `isOperational` flag, and optional structured `details`.

Purpose: All later v2 phases (DIAG classifier in Phase 8, checkpoint/contract errors in Phases 9/12) build their typed error taxonomy on this base. It must admit future codes (`retry-exhausted`, `checkpoint-conflict`, `contract-violation`) without weakening existing narrow `code` unions â€” but those future codes are NOT created here.

Output: `src/errors/app-error.ts` (the generic abstract base + barrel export) and its colocated unit test. No existing files are modified in this plan; subclass re-parenting happens in Wave 2 (plan 07-03).
</objective>
