<objective>
Introduce the structured-logging substrate CORE-02 requires: a `createLogger` factory in a new `src/logging/` directory built on `pino`, with secret redaction that mirrors the existing `redactConfig` posture and an injectable `destination` stream for testability. The factory supports child loggers keyed by `runId` (the wiring into the CLI happens in Wave 2, plan 07-03).

Purpose: This is the emission substrate that DIAG retry events (Phase 8), checkpoint info/error events (Phase 9), and PROG NDJSON progress events (Phase 11) all build on. It must redact secrets by default and stay synchronous (no async transport) so later phases can add an awaited flush (PROG-04) without redesign.

Output: `pino` added to dependencies, `src/logging/create-logger.ts` (factory + `CreateLoggerOptions` type + `REDACT_PATHS`), and a colocated unit test capturing NDJSON via an injected stream. This plan does NOT touch `src/cli.ts` and does NOT change the final JSON summary stdout contract.

Autonomy note: this plan is NOT fully autonomous — it contains a blocking package-legitimacy human-verify checkpoint before the `pnpm add pino` install (security_enforcement = ASVS L2).
</objective>
