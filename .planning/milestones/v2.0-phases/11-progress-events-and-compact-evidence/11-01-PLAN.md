<objective>
Create the opt-in S3 evidence artifact store and its config knob — the durable
write target for the full per-candidate `RunSummary` (PROG-03). This is the
write-once durable surface that subsequent plans (run-once emission, cli wiring)
consume. Mirror the Phase 9 checkpoint store's `create*FromConfig` + injectable
`sender` seam and validated pure object-key helper, but strip every concurrency
mechanism: evidence is write-once per unique `runId`, so there is no read path,
no CAS loop, no `IfMatch`/`IfNoneMatch`, and no merge (D-10).

Purpose: PROG-03 requires a durable S3 object `runs/<runId>/evidence.json`
carrying the full evidence that PROG-02 strips from stdout. This plan builds the
store + key + config in isolation so it can be wired in Wave 2 without it
existing yet blocking the event/projection work.
Output: `src/evidence/` (store, object-key, fixtures, unit + integration tests)
and the `s3.evidencePrefix` Zod knob in `src/config.ts`.
</objective>
