<objective>
Two disjoint, pure seam changes that the Wave-2 run-once/cli wiring consumes:
(1) additively thread `httpStatus` from the failure classifier into the retry
event so PROG-01's `retry` event can carry `attempt`/`httpStatus`/`causeCode`
(D-06, Pitfall 4); (2) add the pure `toCompactSummary` projection +
`CompactRunSummary` type that PROG-02 prints to stdout in place of the heavy
`RunSummary`.

Purpose: PROG-01 requires the `retry` event to expose `httpStatus`, which today
lives on `FailureClassification` but not on `RetryAttemptEvent`. PROG-02 requires
a compact stdout document; `toCompactSummary` is the pure transform that strips
the four heavy arrays. Both are leaf changes with their own tests, runnable
before any orchestration is rewired.
Output: `httpStatus` on `RetryAttemptEvent` (additive, optional);
`CompactRunSummary` type; `toCompactSummary` pure function — all with tests.
</objective>
