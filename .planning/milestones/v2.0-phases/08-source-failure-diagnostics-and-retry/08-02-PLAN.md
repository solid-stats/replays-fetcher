<objective>
Wire the shared classifier and retry wrapper from Plan 01 into the list/detail source adapter (`source-client.ts`), and widen the diagnostic type surface (`discovery/types.ts`) so failures carry the DIAG-01 rich fields. Add Cloudflare status-200 challenge detection in the direct adapter (DIAG-02), enrich `SourceFetchError.details` with identifiers-only evidence (DIAG-01/04), and accept the retry config + `onRetry`/`signal` seam through `SourceClient.fetchText` so the orchestrator (Plan 04) can drive retry attempts and warn events.

Purpose: This is the list/detail half of the "one shared classifier + retry, both adapters" locked decision. It runs in parallel with Plan 03 (the bytes half) because they touch disjoint files.

Output: enriched `SourceFetchError` details, both direct + SSH reads routed through `classifyFailure` + `withRetry`, CF status-200 detection, and the widened `DiscoveryDiagnostic`/`DiagnosticCode` + `SourceClient` types.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external source → source-client.fetchText | Untrusted HTTP responses (status, headers, body), SSH stdout/stderr, and transport errors cross into the adapter. |
| source-client → SourceFetchError.details → diagnostics | Adapter-built error details flow toward operator-visible diagnostics and stderr logs. |
| operator config → base64 SSH positional arg | Source URL is passed to SSH as a base64-encoded positional argument, never interpolated into the shell command. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Information Disclosure | `SourceFetchError.details` on list/detail failures | mitigate | Identifiers-only allowlist (phase, httpStatus, causeCode, causeMessage, url, attempts, cfChallenge); never copy body/bytes/headers-with-tokens/SSH-command into details. No-body-leak unit test (DIAG-04). |
| T-08-02 | Elevation of Privilege / Injection | SSH transport URL handling | mitigate | URL stays base64-encoded into the existing positional `$1` arg decoded with `base64 -d` (unchanged from v1); no string interpolation of URL into the shell command — injection surface is unchanged. |
| T-08-03 | Denial of Service | direct/SSH read retry loop | mitigate | Bounded `attempts` from config threaded through `withRetry`; per-round `sourceTimeoutMs` timeout; full jitter + 30s cap (Plan 01). |
