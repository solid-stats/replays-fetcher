## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| adapter errors → discover/run diagnostics → stdout summary | Enriched error details flow into the operator-visible stdout JSON summary. |
| retry events → runId child logger → stderr | Server-controlled `causeMessage`/`causeCode` flow into stderr NDJSON warn logs. |
| config → sourceRetryAttempts | Operator-supplied attempt count controls the retry loop bound. |

## STRIDE Threat Register (ASVS L2)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Information Disclosure | enriched `DiscoveryDiagnostic` / run summary on stdout | mitigate | Diagnostics carry only allowlisted identifier fields (phase, httpStatus, causeCode, causeMessage, page, url, attempts, cfChallenge); no body/bytes/secret — verified end-to-end by no-body-leak coverage and stdout-shape assertions. |
| T-08-02 | Information Disclosure | retry warn events to stderr | mitigate | Logged via `createLogger` redact paths (s3 keys, ssh command, db url) on the `runId` child; only identifiers logged; cli test asserts no secret in captured stderr. |
| T-08-03 | Tampering | log injection via server-controlled `causeMessage` | mitigate | `causeMessage` emitted as a structured pino JSON value, never interpolated into a format string (warn message is a static string). |
| T-08-04 | Tampering / contract break | stdout JSON summary contract | mitigate | Warn events ride stderr (createLogger default destination), not stdout; existing cli summary-shape assertions run unchanged as a regression guard (CR-01). |
| T-08-05 | Denial of Service | operator-supplied `sourceRetryAttempts` | mitigate | Zod-validated nonnegative int (Plan 01); bounded loop + full jitter + 30s cap make total retry time finite. |
| T-08-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan (08-RESEARCH.md audit N/A); only existing modules + Plan 01-03 symbols. |
</threat_model>
