<objective>
Build the S3 checkpoint store: a `GetObject` read (stream → string → `parseCheckpoint`) and a conditional `PutObject` write (`IfNoneMatch:*` create / `IfMatch:<etag>` update) with a bounded CAS loop that, on `412 PreconditionFailed` (and the sibling `409 ConditionalRequestConflict`), re-reads, merges keeping `max(lastCompletedPage)`, and retries. Mirrors the injectable `sender` seam + `FromConfig` factory of `s3-raw-storage.ts`, widened to `GetObjectCommand`. Proven against MinIO Testcontainers.

Purpose: RESUME-01 (durable single rolling checkpoint object, identifiers-only) and RESUME-02 (conditional-write CAS so concurrent/restarted pods cannot silently clobber a newer checkpoint).
Output: `src/checkpoint/object-key.ts` (slug→key), `src/checkpoint/s3-checkpoint-store.ts` (+ unit + MinIO integration test).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| S3 GetObject body → fetcher | The checkpoint read back is untrusted (corrupt, hand-edited, stale-schema) — parsed via `parseCheckpoint` (degrade). |
| concurrent pods → same S3 key | Two writers racing on `checkpoints/<slug>/latest.json`; the CAS guard is the integrity control. |
| operator prefix/source URL → S3 key | Slug derived from URL must be S3-safe to prevent key-injection. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Information Disclosure | checkpoint object body | mitigate | Body is the identifiers-only `Checkpoint` (Plan 01 allowlist); no bytes/secrets/HTML written. Integration test reads the object back and asserts no body/secret content (ASVS L2 V8.3). |
| T-09-08 | Tampering | concurrent checkpoint clobber (RESUME-02 core security property) | mitigate | Conditional writes (`IfNoneMatch:*` create / `IfMatch:<etag>` update); 412/409 → re-read + `mergeCheckpoints` keeping `max(lastCompletedPage)` so a newer checkpoint is never silently overwritten. Proven by the MinIO 412 integration test (ASVS L2 V11 / optimistic-concurrency integrity). |
| T-09-02 | Tampering | corrupt/stale checkpoint read | mitigate | `read` delegates to `parseCheckpoint` → corrupt body returns undefined (clean-start), never throws (RESUME-03). |
| T-09-09 | Denial of Service | unbounded CAS retry under contention | mitigate | The 412 re-read loop is bounded (named const, ~5 rounds) with full-jitter backoff; on exhaustion it throws `CheckpointConflictError` rather than looping forever (ASVS L2 V11.1 resource bounds). |
| T-09-05 | Tampering | source-URL-derived S3 key | mitigate | `toCheckpointObjectKey` constrains the slug to `[a-z0-9._-]` and validates the final key against an S3-safe pattern, throwing on violation. |
| T-09-SC | Tampering | npm/pip/cargo installs | accept | No new package installs — `@aws-sdk/client-s3` and `@testcontainers/minio` are already present (09-RESEARCH "No new dependencies"). |
</threat_model>
