  <behavior>
    - Happy path: given a SourceClient returning valid list HTML (page 1), valid detail HTML (with #filename), and valid JSON at the raw URL, returns `{ ok: true, sample, warnings: [] }` where `sample` records `listPageUrl`, `detailUrl`, and `rawUrl`.
    - Swap regression: when the raw URL returns HTML (not JSON), `JSON.parse` throws → returns `{ ok: false, reason: "contract_broken", message, warnings }`.
    - rawUrl identity: the probed raw URL MUST equal `toRawReplayUrl(filename, detailUrl)`; if a regression changed derivation so it no longer points at `/data/<filename>.json`, that is reason `contract_broken`.
    - Empty list page (no rows) → warning `{ code: "empty_list_page" }`, `ok: true` (source reachable, just no sample).
    - First row missing `source.url`/`externalId` → warning `missing_external_id`, `ok: true`.
    - Detail HTML missing filename (`extractFilenameFromDetailHtml` → undefined) → warning `missing_filename`, `ok: true`.
    - SourceFetchError with `.code === "source_transient"` or `"rate_limited"` → `{ ok: false, reason: "source_unreachable", ... }`.
    - SourceFetchError with `.code === "source_unavailable"`: re-classify via `classifyFailure({ httpStatus: details.httpStatus, error })`; `permanent` → `contract_broken`, else `source_unreachable`.
    - Any non-SourceFetchError thrown → reason `contract_broken` (unexpected structural problem).
  </behavior>
  <action>Create `src/contract-check/contract-check.ts` as a pure-function module (analog: src/source/classify-failure.ts shape; PATTERNS service section). Flow (RESEARCH "contract-check data flow"): (1) `fetchText(sourceUrl)` for page 1 — NO options object (single try, per Pitfall 1); run `extractReplayRows(html, 1, sourceUrl)`; empty → push `empty_list_page` warning and return ok:true with the listPageUrl-only sample. (3) `fetchText(detailUrl)`; `extractFilenameFromDetailHtml`; undefined → push `missing_filename` warning, return ok:true. Error handling: wrap each `fetchText` so a thrown `SourceFetchError` is mapped per the classifier rules in `<behavior>` (RESEARCH Pitfall 5 / Code Examples "classifyContractError").
