## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | `src/storage/s3-raw-storage.ts` | mitigate | Require matching `ContentLength` and `Metadata.sha256` before skip; mismatch returns conflict and blocks PUT. |
| T-03-05 | Repudiation | raw object metadata | mitigate | Write checksum metadata and return evidence containing bucket, key, checksum, byte size, and fetched timestamp. |
| T-03-06 | Denial of service | S3 SDK failures | mitigate | Convert expected HEAD/PUT errors into structured `failed` evidence instead of crashing the whole storage pipeline. |
| T-03-07 | Information disclosure | S3 config | mitigate | Adapter receives credentials but result evidence never includes access keys or secret keys. |
</threat_model>
