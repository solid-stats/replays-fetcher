<objective>
Split `src/discovery/source-client.ts` (534L) into the parent plus two cohesive same-band siblings in `src/discovery/`, and remove its `oxlint-disable max-lines` suppression. Pure structural move: no logic, identifier, or signature change. Implements SPLIT-03 [src: 22-RESEARCH.md#SPLIT-03].

Purpose: drop the parent and every sibling under the 300-line threshold with the suppression gone. THE non-trivial detail: the error-builder helpers reference `SourceFetchError`, and the parent's adapter factories reference the error builders. If the error class stayed in the parent, the sibling-imports-parent + parent-imports-sibling edges form a cycle that fails `no-circular`. Mitigation (Pitfall 2): physically move `SourceFetchError` INTO `source-client-error.ts` and re-export it from the parent, so the dependency is strictly parent to sibling with no back-edge, and `./source-client.js` remains the unchanged import path for `discover.ts` and `contract-check.ts`.

Output: `src/discovery/source-client.ts` (~230L), `src/discovery/source-client-error.ts` (~270L), `src/discovery/source-client-retry.ts` (~70L).
</objective>
