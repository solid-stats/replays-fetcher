<objective>
Rewrite the staging repository's benign-duplicate detection from insert-and-catch-23505 to a
targeted `INSERT ... ON CONFLICT (checksum, object_key) DO NOTHING RETURNING id`, and add a lean
`existsBySourceIdentity` existence check that the watch pre-fetch gate (Plan 03) will call. The
conflicting-duplicate manual-review path (same source_replay_id, different checksum -> `conflict`)
stays exactly as-is because the `(source_system, source_replay_id)` constraint is deliberately NOT
in the `ON CONFLICT` target [src: 24-CONTEXT.md#DEDUP-03]. Implements DEDUP-02 and DEDUP-03.

Purpose: ends the postgres duplicate-key ERROR log spam (exception-as-control-flow) for the benign
re-stage, and provides the pre-fetch existence primitive — without touching `server-2`'s
conflict-routing contract.
Output: an updated `postgres-staging-repository.ts` plus expanded unit + testcontainers integration
tests proving the benign path is quiet and the conflict path is NOT swallowed.
</objective>
