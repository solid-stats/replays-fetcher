    ШАГ 0 (источник истины): прогнать `pnpm run lint` (или
    `node_modules/.bin/oxlint --config .oxlintrc.json src`) и собрать ЖИВОЙ список всех
    `func-style` нарушений (файл+строка). Этот live-список — единственный источник work-list;
    НЕ полагаться на 16-01-SUMMARY.md. Сверить категории/порядок величин с

    в `const name = (args): Ret => { ... }`. Сохранять `export` (`export function` →
    `export const name = ... =>`), `async` (`async function` → `const name = async (...) =>`),

    КРИТИЧНО про hoisting: shared preset держит `typescript/no-use-before-define: error` с
    допуском функций в eslint-варианте off — после конвертации в const стрелки теряют
    hoisting. Если файл имел top-level функцию, используемую выше по тексту (фабрики
    `createDirectSourceClient`/`createSshSourceClient` в source-client.ts и
    replay-byte-client.ts — spike показал no-use-before-define именно там), ПЕРЕУПОРЯДОЧИТЬ
    объявления так, чтобы определение шло до использования (move declaration up — как
    советует сам oxlint help). НЕ менять сигнатуры, имена, экспортируемую поверхность.
    Запрещено добавлять `// oxlint-disable func-style`. Работать пакетами по директориям;
    после каждой директории прогонять `pnpm test` для затронутых модулей.
