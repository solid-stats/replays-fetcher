<objective>
Rewire the run-once page loop into a runtime-discovered, parallel, throttled cycle (RANGE-01/02/03/06,
and the RANGE-04 list-page floor): unbounded stop-on-empty loop with the optional cap; classify-before-stop
so a transient failure is never silent truncation; per-page store+stage fan-out over the single shared
`p-limit` limiter gathered with `Promise.allSettled` and re-ordered by candidate index before the
checkpoint; AIMD throttle resizing the shared limiter on `rate_limited`/clean pages; and the `createPacer`
floor between sequential list pages replacing the blanket 2000ms delay.

Purpose: This is the phase's hardest correctness work — the checkpoint-after-page ordering (Phase 9) and
the never-mistake-transient-for-end-of-corpus guard (the literal 2026-05-11 failure) both live here.
Output: rewritten `run-once.ts` loop/`processPage` + retired blanket delay in `discover.ts`, with the
existing colocated tests extended for every new branch.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| source page result → stop decision | A `!ok`/empty page could be misread as end-of-corpus (silent truncation) |
| run loop → upstream source | Concurrency + pacing govern how hard the run hits the Cloudflare-fronted source |
| candidate fan-out → store/stage | Operational failures vs programmer-error rejections cross the gather boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-08 | Tampering / data loss (silent truncation) | stop-on-empty vs classifier ordering | mitigate | Classify FIRST via `deriveSourceFailure`; only `ok && candidates.length === 0` is end-of-corpus → `complete`. A `transient`/`rate_limited` page is `resumable`, never `complete` — closes the 2026-05-11 corpus-truncation risk. Covered by the no-silent-truncation test. |
| T-10-09 | Denial of Service (against the source) | retry storm under throttle + backoff | mitigate | The throttle shrinks the single shared limiter's `.concurrency` on `rate_limited`; per-request backoff stays in `withRetry`; the two never compound (Pitfall 2). The pacer enforces a floor between list pages. |
| T-10-10 | Denial of Service / resource exhaustion | unbounded stop-on-empty loop never terminating | mitigate | Optional `REPLAY_SOURCE_MAX_PAGES` safety-valve cap; a `permanent`-classified page stops the loop; only an `ok` zero-row page stops `complete`. README documents an operator-set cap for safety. |
| T-10-11 | Tampering | dropped/duplicated candidate evidence under parallel gather | mitigate | `Promise.allSettled` + re-order by captured candidate index before tally/checkpoint gives deterministic, race-free evidence ordering; a rejected settle is rethrown (programmer error), never silently dropped. |
| T-10-12 | Information Disclosure | per-page rate line / diagnostics | accept | Identifiers-only (page number, counts) — no URLs-with-userinfo (run-once already strips via `sanitizeSourceUrl`), no bytes/HTML/secrets; consistent with DIAG-04. |
</threat_model>
