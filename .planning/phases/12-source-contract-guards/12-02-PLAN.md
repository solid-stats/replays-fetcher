<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI argv → command handler | Operator invocation; no untrusted args beyond the configured source |
| handler → source (read-only) | Derived from Zod-validated `SourceConfig.sourceUrl` — not attacker-controlled (low SSRF) |
| result → stdout | Serialized result must carry no secrets/source bodies |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-03 | Information disclosure | `writeJson(result)` on stdout | mitigate | The Plan-01 result carries only URLs + boolean/reason fields, never response bodies or secrets (DIAG-04); the source URL is operator-configured and already surfaced by `check`/`discover`, so emitting it is consistent with existing posture |
| T-12-04 | Spoofing/SSRF | probed URL origin | accept | The sampled URL derives from the Zod-validated configured `sourceUrl` (config.ts `z.url()`), not from user input — no attacker-controlled fetch target |
| T-12-05 | Elevation/boundary breach | contract-check handler | mitigate | Handler wires only `loadSourceConfig`+`createSourceClient`+`runContractCheck`; GUARD-04 spy test asserts no S3/staging factory is constructed (no-write guarantee, AGENTS boundary) |
| T-12-SC | Tampering | npm/pip/cargo installs | accept | No new packages this phase (RESEARCH Package Legitimacy Audit: none) |
</threat_model>
