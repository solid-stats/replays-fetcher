<objective>
Promote the run-once log emission from the Phase-7 `log.debug` stub and the
Phase-10 minimal `emitPageRateLine` to the real PROG-01 lifecycle taxonomy as
greppable pino NDJSON on stderr, and wire the opt-in evidence write
(log-and-continue) into the run loop.

Emit `run_start` (info), `page_complete` (info, counts+rates),
`page_failed`/`source_unavailable` (error), and `run_complete`/`run_partial`
(info/warn) — each a structured object with a stable `event:<name>` discriminator
and a static message (D-03/D-04). Reuse `derivePagesPerMinute` as the single rate
source (D-05), `deriveSourceFailure` for the failure fields, and `deriveRunStatus`
for the final status. Add an injectable evidence-store seam + a dev-only
evidence-file writer seam; the write is the checkpoint try/catch → warn pattern
(D-12) — it never fails the run.

Purpose: PROG-01 makes a ~786-page run followable in real time; PROG-03 persists
the full evidence durably without putting it on stdout. The `retry` (warn) event
already flows through the existing `buildRetryWarnEmitter → onRetry → withRetry`
seam (now carrying `httpStatus` from Plan 02) — this plan does NOT re-emit retry,
it only adds the page/lifecycle events and the evidence write.
Output: the lifecycle event taxonomy in `run-once.ts` + the opt-in evidence write.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| run loop → stderr NDJSON | per-page/lifecycle events cross into a durable operator log stream |
| in-memory RunSummary → opt-in S3 object / local file | the full summary is serialized to durable surfaces |
| operator-supplied --evidence-file path → local disk | first local-disk write in the project |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-06 | Information Disclosure | lifecycle event payloads | mitigate | Every event is a structured identifiers-only object (runId/page/counts/classification/code/status); the sourceUrl is the `sanitizeSourceUrl` userinfo-stripped slug; the message is a static string (no interpolation). pino `REDACT_PATHS` cover any stray secret shape. The exhaustive no-leak assertion over events lands in Plan 05 (T-11-09). |
| T-11-07 | Denial of Service / boundary | opt-in evidence S3/file write | mitigate | The write is gated by `emitEvidence`/`evidenceFile` (default off) and is log-and-continue — it never fails the run, never changes the exit code, and adds no checkpoint/staging/raw mutation. The S3 object is the only new durable write surface (shared bucket/credentials). |
| T-11-08 | Repudiation | dropped final event lines | accept (this plan) | runOnce emits events synchronously through the stderr logger; the awaited `flush()` guarantee (PROG-04) is owned by the cli action in Plan 04 (T-11 flush) — runOnce does not call `process.exit()` and does not own the flush. |
| T-11-SC | Tampering | npm/pip/cargo installs | mitigate | No package installed in this phase. The dev-only file write uses `node:fs` (stdlib) injected as a seam; no new dependency. |
</threat_model>
