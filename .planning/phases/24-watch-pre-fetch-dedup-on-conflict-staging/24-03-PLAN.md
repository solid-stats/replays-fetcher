<objective>
Wire the watch-only pre-fetch dedup gate into the shared `ingestPage` helper behind an explicit
per-call `prefetchDedup` flag that ONLY watch's `runCycle` sets, so the DRY core stays shared while
run-once remains byte-for-byte unchanged (its golden e2e oracle never moves)
[src: 24-RESEARCH.md#Pattern-3]. The skip fires only on a trustworthy non-empty `externalId` whose
staging row already exists (the cannot-miss invariant), tallies a distinct `skippedBySourceId`, and
the golden-watch oracle is FLIPPED from "re-download every cycle" to "no byte fetch on cycles >=2"
[src: 24-CONTEXT.md#DEDUP-01]. Implements DEDUP-01 end-to-end and the behavior-change oracle flip;
relies on Plan 01 (existsBySourceIdentity) and Plan 02 (the counter).

Purpose: collapse a no-new-replay watch cycle to a single page-1 listing fetch without ever silently
dropping a genuinely-new replay [src: 24-CONTEXT.md#risk_summary].
Output: an updated `ingest-page.ts` (gate + skip discriminant + tally), `watch-loop.ts` (sets the
flag, threads sourceSystem + the skip count into the summary), the cannot-miss property test, and the
flipped golden-watch oracle.
</objective>

| Boundary | Description |
|----------|-------------|
| discovered candidate -> pre-fetch skip decision | An untrusted externalId from the source HTML decides whether bytes are fetched; a wrong skip silently drops a new replay |
| fetcher -> PostgreSQL | existsBySourceIdentity issues a parameterized read keyed on (sourceSystem, externalId) |
