<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external replay source -> fetcher | Untrusted replay bytes enter the process. |
| candidate source URL -> byte client | Source URL from discovery drives byte fetch location. |
| byte client -> S3 storage adapter | Raw bytes and computed identity move into storage adapter. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-08 | Spoofing | `ReplayCandidate.source.url` | mitigate | Reuse Phase 2 same-origin/source transport assumptions and test that candidate URL is the byte-fetch target. |
| T-03-09 | Tampering | `storeRawReplay` | mitigate | Compute checksum from fetched bytes before deriving object key or calling storage. |
| T-03-10 | Denial of service | byte fetching | mitigate | Reuse configured timeout/transport failure handling and return structured `failed` evidence. |
| T-03-11 | Information disclosure | failure evidence | mitigate | Failure evidence must include category/message only; never include credentials or raw replay byte payloads. |
</threat_model>
