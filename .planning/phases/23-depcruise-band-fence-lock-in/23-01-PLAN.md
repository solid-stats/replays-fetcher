<objective>
Encode the fetcher's five-band ingest architecture as eight dependency-cruiser `forbidden` rules
at severity `error` inside `verify`, and prove with a planted-violation test that each fence
actually fires. This is the milestone's most important sequencing invariant — fences are enforced
LAST, as a NO-OP lock-in: Phases 19–22 already left the tree fence-clean (research PROVED depcruise
exit 0 on a scratch config with all 8 rules), so this phase locks in completed work and prevents
future drift, never wedges an in-flight move.

Purpose: harden the ingest boundary (PG/S3 write-scope, no-parser, downward-only banding) as a
static `verify` gate so a stray `pg`/S3 client, an OCAP parser, or an upward cross-band import is
caught at CI time instead of leaking into production.

Output: the 8 fences in `.dependency-cruiser.cjs` (the old `no-commands-to-storage-direct` warn
removed), plus `src/depcruise-fences.test.ts` proving all 8 fire.
</objective>
