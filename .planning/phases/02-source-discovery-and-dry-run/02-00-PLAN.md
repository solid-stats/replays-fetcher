# Plan 02-00: Dry-Run Happy Path

<objective>
As an operator, I want `replays-fetcher discover --dry-run` to emit a deterministic JSON candidate report from a source adapter, so that I can inspect replay candidates without mutating S3, staging, parser, or backend state.
</objective>

<threat_model>
## Threat Model

- T-02-01: Dry-run accidentally performs mutating ingest work. Mitigation: implement discovery through a source-client interface only; no S3, PostgreSQL, filesystem replay-list, parser artifact, or `run-once` code is introduced in this plan.
- T-02-05: Report shape drifts before Phase 3/4 can reuse it. Mitigation: define typed report/candidate/diagnostic structures and assert exact JSON fields in tests.
</threat_model>

<action>
Create `src/discovery/types.ts` with exported TypeScript types:
- `DiscoveryMode = "dry-run"`
- `DiagnosticSeverity = "info" | "warning" | "error"`
- `DiagnosticCode = "malformed_row" | "missing_filename" | "duplicate_filename" | "changed_metadata" | "source_unavailable" | "rate_limited"`
- `ReplayCandidate` with `identity.filename`, `source.url`, optional `source.externalId`, optional `source.page`, optional `metadata.missionText`, optional `metadata.world`, optional `metadata.serverId`, and optional `metadata.discoveredAt`.
- `DiscoveryDiagnostic` with `code`, `severity`, `message`, optional `sourceUrl`, optional `page`, optional `candidateIndex`, and optional `externalId`.
- `DiscoveryReport` with `ok`, `mode`, `sourceUrl`, `generatedAt`, optional `maxPages`, `counts`, `candidates`, and `diagnostics`.
- `SourceClient` with `fetchText(url: URL): Promise<string>`.

Create `src/discovery/discover.ts` with `discoverReplaysDryRun(options)` that accepts `sourceUrl`, `sourceClient`, optional `generatedAt`, optional `maxPages`, and returns a `DiscoveryReport`. For this first slice, support a source client that returns a JSON fixture shaped as:
`{"candidates":[{"filename":"replay-a.json","url":"https://example.test/replays/100","externalId":"100","missionText":"sg@test","world":"Altis","serverId":1}]}`
Map that fixture into a report with `ok: true`, `mode: "dry-run"`, `counts.discovered: 1`, `counts.candidates: 1`, `counts.diagnostics: 0`, and no mutating side effects.

Add `tests/discovery.test.ts` proving this function returns one candidate with `identity.filename === "replay-a.json"`, `source.url === "https://example.test/replays/100"`, `source.externalId === "100"`, and zero diagnostics.
</action>
