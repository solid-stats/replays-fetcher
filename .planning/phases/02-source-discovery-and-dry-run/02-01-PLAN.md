# Plan 02-01: Source HTML Discovery and Stable Identity

<objective>
As an operator, I want dry-run discovery to parse replay list/detail pages into stable filename-based candidates, so that repeated dry runs over the same source evidence produce the same candidate identities.
</objective>

<threat_model>
## Threat Model

- T-02-02: Candidate identity is canonicalized or inferred incorrectly. Mitigation: preserve the source filename exactly after minimal trim/non-empty validation and test `#filename` precedence over `body[data-ocap]`.
- T-02-05: Live-source assumptions are trusted without fixtures. Mitigation: fixture all list/detail parsing behavior and avoid live network in automated tests.
</threat_model>

<action>
Create `src/discovery/html.ts` with pure parsing helpers:
- `extractReplayRows(html: string, page: number, sourceUrl: URL)` returns row observations from `.common-table > tbody > tr`.
- For each valid row, read the first anchor `href` and text as `source.url` and `metadata.missionText`.
- Read table cell index 1 as `metadata.world`.
- Read table cell index 2 as numeric `metadata.serverId` when parseable.
- Derive `source.externalId` from links matching `/replays/{id}`.
- Resolve relative replay links against `sourceUrl`.
- `extractFilenameFromDetailHtml(html: string)` returns the `value` attribute of `#filename` when non-empty; otherwise returns `body[data-ocap]` when non-empty; otherwise returns `undefined`.

Use conservative parsing. Do not apply old parser include/exclude filtering, snake-case mission rewrites, file saving, or raw replay downloads.
</action>
