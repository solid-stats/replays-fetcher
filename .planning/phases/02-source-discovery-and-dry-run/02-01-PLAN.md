# Plan 02-01: Source Transport, HTML Discovery, and Stable Identity

<objective>
As an operator, I want dry-run discovery to fetch the source directly or through a simple SSH-backed transport and parse list/detail pages into stable filename-based candidates, so that Cloudflare allowlisting can be used without rebuilding the old relay path.
</objective>

<threat_model>
## Threat Model

- T-02-02: Candidate identity is canonicalized or inferred incorrectly. Mitigation: preserve the source filename exactly after minimal trim/non-empty validation and test `#filename` precedence over `body[data-ocap]`.
- T-02-05: Live-source assumptions are trusted without fixtures. Mitigation: fixture all list/detail parsing behavior and avoid live network in automated tests.
- T-02-06: SSH access turns into an overbuilt relay subsystem. Mitigation: implement only a narrow optional source-client transport that invokes a configured SSH fetch command; do not add services, daemons, or persistent relay state.
</threat_model>
