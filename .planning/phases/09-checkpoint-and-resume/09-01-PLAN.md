| S3 checkpoint object → fetcher (parse) | Checkpoint bytes read back from S3 are untrusted input (could be corrupt, hand-edited, or from a different schema version) — parsed by `parseCheckpoint` in this plan. |
| error `details` → logs/stdout (downstream) | `CheckpointConflictError.details` flows into logs; must carry no secrets/bytes. |
