<objective>
Заменить `tsc`-emit на tsdown 0.22.2: единый entry `src/cli.ts` → один файл `dist/cli.mjs` (ESM, runtime deps externalized), удалить `tsconfig.build.json`, оставить `tsc --noEmit` как typecheck, обновить `bin` и Dockerfile, и доказать что собранный CLI РЕАЛЬНО запускается в чистом Docker-образе через smoke-run команды `check`.

Purpose: tsc-tree-emit заменяется на single-file bundle (BLD-01); Docker smoke — единственный runtime gate, доказывающий что externalized-deps резолвятся в проде (BLD-02). Зелёная сборка сама по себе недостаточна — главный pitfall tsdown это «green build, broken runtime».

Output: tsdown@0.22.2 devDep; обновлённый `build` script + `bin`; удалённый `tsconfig.build.json`; обновлённый Dockerfile; build-output `dist/cli.mjs`; записанный в SUMMARY вывод Docker smoke.

Hard invariant: build/emit swap ONLY. Никаких изменений в `src/`, никакого lefthook/CI (Phase 18). Runtime-поведение CLI (команды, флаги, exit-коды, JSON-summary) байт-идентично — bundle это просто другой emit того же `src/cli.ts`.
</objective>
