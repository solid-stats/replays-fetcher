<objective>
Capture the discovery listing's "Game date" cell and thread it through the existing optional-timestamp
pipeline: parse it to a UTC ISO string in the discovery HTML parser (DISC-01), and wire it as a
FALLBACK source for the canonical staging `replayTimestamp` when the source filename carries no
timestamp (DISC-02). This is the second and final intentional behavior change of v3.1.

Purpose: server-2 consumes the staging `replayTimestamp` column (`resolveReplayTimestamp` ->
`replays.replay_timestamp`, indexed, read by web/stats). Replays whose filename lacks a timestamp
currently fall to server-2's `deriveReplayTimestampFromSourceId` epoch fallback; capturing the
listing game-date gives web/stats a real game-date for those replays, and populates
`promotion_evidence.discoveredAt` as audit lineage for ALL replays.

Output: a `parseGameDateToUtcIso` helper + `cells[3]` read in `src/discovery/html.ts`; a
filename-primary / listing-fallback resolution in `src/staging/payload.ts`; new unit cases in
`html.test.ts` and `payload.test.ts`; and a flipped golden-e2e oracle asserting the concrete
`discoveredAt` value.
</objective>
